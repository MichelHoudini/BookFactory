# BF-002 — Translation Unit Generator

## Goal

Split the canonical Document into Translation Units suitable for LLM translation.

Each Translation Unit should contain approximately 900 words while preserving the original document order.

---

## Input

One canonical Document.

---

## Responsibilities

Implement:

- Receive a Document.
- Count words.
- Group consecutive Blocks into Translation Units.
- Never split a Block.
- Preserve reading order.
- Keep each Translation Unit close to 900 words.
- Store the ordered list of Block IDs contained in each Translation Unit.

---

## Output

An ordered collection of Translation Units.

Example:

Unit 1
- Block 1
- Block 2
- Block 3

Unit 2
- Block 4
- Block 5

---

## Out of Scope

Do NOT implement:

- Translation
- OpenAI
- Cache
- Retry
- Prompt loading
- EPUB generation
- Validation
- Reconstruction

---

## Acceptance Criteria

The Translation Units preserve document order.

No Block is split.

The resulting units are approximately 900 words.

The implementation follows PROJECT.md and docs/ARCHITECTURE.md.

Only the minimum code required for BF-002 is implemented.
